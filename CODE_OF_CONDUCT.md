# Code of Conduct

## Our Pledge

This is a small open-source project built and maintained by one developer.
Everyone who participates — through issues, PRs, or discussions — is expected
to keep interactions respectful, focused, and constructive.

---

## Standards

**Acceptable behavior:**

- Respectful, constructive feedback
- Reporting bugs clearly with enough detail to reproduce them
- Suggesting features with reasoning
- Helping others in issues with knowledge you have
- Accepting that PRs may be closed or changed during review

**Unacceptable behavior:**

- Harassment, insults, or personal attacks
- Demanding features, merges, or responses on a timeline
- Spamming issues or PRs
- Requesting or pressuring for direct repository access
- Sharing others' personal information without consent
- Any discriminatory language or behavior

---

## Maintainer Rights

This project is maintained by [@Venkatesh-6921](https://github.com/Venkatesh-6921).

The maintainer has the right to:

- Close any issue or PR without explanation
- Reject contributions that don't fit the project direction
- Block users who repeatedly violate this code of conduct
- Change project direction at any time

**Direct repository access will not be granted to external contributors.**
All contributions go through the fork → PR → review → merge workflow.
Only the maintainer merges pull requests.

---

## Reporting Issues

If someone is violating this code of conduct, open an issue with the
`conduct` label or contact the maintainer directly through GitHub.

Reports will be reviewed and acted on at the maintainer's discretion.

---

## Scope

This Code of Conduct applies to:

- GitHub Issues
- Pull Requests
- Commit messages and PR descriptions
- Any communication in this repository's spaces

---

## Enforcement

Violations may result in:

1. A warning comment on the issue/PR
2. Closing the issue/PR without further engagement
3. Blocking from the repository
